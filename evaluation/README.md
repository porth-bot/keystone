# Keystone — Evaluation (Layer-1 BKT validation)

Layer-1 validation for Keystone's **per-skill 4-parameter Bayesian Knowledge
Tracing (BKT)** model. Everything here runs on the **Python standard library
alone** (no numpy / scikit-learn / pyBKT required), is **deterministic**
(seeded), and makes **no network calls**.

> **Honesty note.** The default pipeline evaluates a **SYNTHETIC** cohort
> generated from the BKT process itself. It is **not** trained or validated on
> real students. Only synthetic numbers are checked in. See `model_card.md`.

## Files

| File | What it is |
|------|------------|
| `fit_bkt.py` | Generates a synthetic cohort and fits per-skill `{L0,T,G,S}` via EM (Baum-Welch); writes `parameters_fit.json`. Optional `--data` real-data path. |
| `compare_baselines.py` | Baseline predictors (`majority`, `previous_answer`) as an importable module; runnable for a small demo. |
| `evaluate.py` | Held-out next-answer evaluation (BKT vs. baselines); writes `results.json`. |
| `parameters_fit.json` | Fitted parameters (generated by `fit_bkt.py`). |
| `results.json` | Actual computed metrics (generated by `evaluate.py`). |
| `model_card.md` | Honest model card with the real numbers and limitations. |

## How to run

```bash
python3 fit_bkt.py      # fit params  -> parameters_fit.json
python3 evaluate.py     # evaluate    -> results.json
```

`evaluate.py` loads `parameters_fit.json` if present, and otherwise fits on the
fly, so it also works standalone. `compare_baselines.py` can be run directly for
a quick demo of the baseline predictors.

Useful flags for `fit_bkt.py`: `--students N`, `--opps K`, `--seed S`,
`--out PATH`.

## What the numbers mean

- **ROC AUC** — tie-corrected Mann-Whitney rank AUC (stdlib). Higher is better.
  The `majority` baseline is constant, so its AUC is exactly 0.5.
- **Accuracy** — fraction correct at threshold 0.5. Higher is better.
- **Brier score** — mean squared error of the predicted probability. Lower is
  better.

On the synthetic held-out set BKT beats both baselines; the exact figures live
in `results.json` and are reproduced in `model_card.md`. The fit also recovers
the generating parameters closely (see the fitted-vs-true table printed by
`fit_bkt.py`), which sanity-checks the estimator.

## Optional: real-data path (ASSISTments)

You can optionally fit on a real, public dataset instead of the synthetic
cohort. **This repo does not redistribute any student data**, so you download it
yourself. A good choice is the **ASSISTments 2009-2010 "skill-builder"** set —
a public, de-identified collection of student math responses. Get it from the
ASSISTments data page (`https://sites.google.com/site/assistmentsdata/home`) or
a public Kaggle mirror (search "ASSISTments 2009 2010 skill builder").

The `--data` path expects a CSV with at least a **student id**, a **skill id**,
and a **0/1 correctness** column (an optional order column sorts attempts in
time):

```bash
python3 fit_bkt.py --data path/to/skill_builder_data.csv
```

Override column names if your file differs:

```bash
python3 fit_bkt.py --data data.csv \
    --student-col user_id --skill-col skill_id \
    --correct-col correct --order-col order_id
```

If the file is missing, the script prints these instructions and exits cleanly
(exit 0) **without fabricating any numbers**. No ASSISTments results are checked
into this repository.
