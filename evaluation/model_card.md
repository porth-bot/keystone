# Model Card: Keystone BKT Diagnostic

## Overview

**Model.** Per-skill **4-parameter Bayesian Knowledge Tracing (BKT)**. Each
skill is a two-state hidden Markov model (mastery: unknown / known) with
parameters:

| Param | Meaning |
|-------|---------|
| `L0`  | P(skill already known at first opportunity) — prior |
| `T`   | P(unknown → known) between opportunities — learn/transition |
| `G`   | P(correct \| not known) — guess (constrained ≤ 0.30) |
| `S`   | P(incorrect \| known) — slip (constrained ≤ 0.30) |

On each observation the belief `L` is updated Bayesianly and then advanced by
the learning transition:

```
p_correct = L*(1-S) + (1-L)*G
posterior | correct   = L*(1-S) / p_correct
posterior | incorrect = L*S / (L*S + (1-L)*(1-G))
L_new = posterior + (1-posterior)*T
```

Parameters are fit per skill with EM (Baum-Welch) from response sequences only.

## Intended use

A **hackathon diagnostic demo**: estimate a learner's mastery of calculus
prerequisites (limits → continuity → derivatives → chain rule → integrals) from
their answers, to drive next-question selection and feedback. Not intended for
grading, placement, or any high-stakes decision.

## Training / evaluation data

> **This model was evaluated on a SYNTHETIC cohort. It was NOT trained or
> validated on real calculus students.**

The default in-repo pipeline:

1. Generates a synthetic cohort of **600 students × 5 skills × 10 attempts**
   from the BKT generative process using fixed ground-truth parameters
   (deterministic, seeded).
2. Fits `{L0, T, G, S}` back **from the observed responses only** — the fitter
   never sees the hidden mastery states or the true parameters.
3. Holds out each sequence's final attempt and predicts it from the preceding
   attempts.

**Parameter recovery (synthetic).** Mean absolute error across all 20 fitted
parameters vs. the generator: **0.0099** — the EM fitter recovers the generating
parameters closely, which is a sanity check on the estimator, not a claim about
real learners.

## Metrics (actual computed numbers)

Held-out next-answer prediction on the synthetic cohort
(`n = 3000` held-out attempts, `base_rate = 0.6191`). Numbers are copied from
`results.json` as computed; nothing is hand-tuned.

| Model | ROC AUC | Accuracy @0.5 | Brier |
|-------|:-------:|:-------------:|:-----:|
| **BKT** | **0.7403** | **0.8420** | **0.1232** |
| majority (base rate) | 0.5000 | 0.8107 | 0.1902 |
| previous_answer | 0.6702 | 0.7837 | 0.2163 |

Higher AUC/accuracy is better; lower Brier is better. On this synthetic set BKT
beats both baselines on all three metrics. AUC uses the tie-corrected
Mann-Whitney rank statistic; the `majority` predictor is constant, so its AUC is
exactly 0.5 by construction.

## Limitations

- **Synthetic data.** Every number above comes from data generated by the same
  BKT process being fit. That the model predicts BKT-generated data well does
  not establish that it predicts real students well. Real learners violate BKT
  assumptions (no forgetting, single latent skill per item, stationary
  guess/slip).
- **Small scale.** 600 simulated students, 5 skills, 10 attempts each.
- **No real-student validation.** No real classroom or platform data was used.
- **Hand-designed prerequisite graph.** The calculus skill chain is authored by
  us, not learned or empirically validated.
- **Identifiability caps.** `G` and `S` are capped at 0.30 to keep the fit
  identifiable; the true guess/slip of any real item could exceed that.

## Real-data validation (optional — NOT run here)

The scripts include an optional path to fit and sanity-check on a real,
user-supplied dataset (e.g. the public, de-identified **ASSISTments 2009-2010
skill-builder** set). This repository does **not** redistribute any student data,
and **this path has not been run** — no ASSISTments numbers appear anywhere in
this card or in `results.json`.

To run it yourself, download the dataset and:

```
python3 fit_bkt.py --data path/to/skill_builder_data.csv
```

If the file is absent the script prints download instructions and exits cleanly
without producing any numbers. See `README.md` for details.

## Reproducibility

Deterministic and seeded (`seed = 20260717`). Standard library only — no numpy,
scikit-learn, pyBKT, or network access required.

```
python3 fit_bkt.py     # fit -> parameters_fit.json
python3 evaluate.py    # evaluate -> results.json
```
